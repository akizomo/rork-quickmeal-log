📝 デザインノート

  • 背景: 実際のquick log画面 (button grid + ringなど) を blur+dim で表示。「これからここを使うんだ」感を残す
  • overlay構造: フルスクリーンモーダル型 (Duolingo style)
    • 上部: ✕ (close) + プログレスバー (1/1 = full) + 右余白
    • 本体: 白カード (rounded top corners、scrollable) — title/sub/infographic/footnotes
    • 下部固定エリア: ← + ドットインジケータ + → の navigation row + full-width pill CTA「わかった」 (黒地白文字)。border-topで本体カードと視覚分離
  • 下部navigation row: 単一stopなので ← / → はdisabled (opacity 0.25)。ドットは1個 active。将来複数stop追加時もこの構造で対応可能
  • レイアウト変更理由: 旧 bottom sheet (78% 高) より読みやすく、モバイルUIパターンとして馴染みあり (Duolingo型 explanation modal)
  • プログレスバー: 単一stop だが「現在地が見える」ため残置。将来 stop 増えた場合も同じ構造で対応可
  • infographic: 各bucket横棒バー (modal-set range)、●はdefault Identityの値。PFCタグはラベル真下に固定列で揃える (列ぶれなし)
  • 削除した情報: default Identity名 (「ご飯1杯/食パン1枚」等) — dotがdefault値を示すので冗長と判断。情報量削減で視認性↑
  • 削除した情報 (続): scale-header (0/300 kcal) も削除。バー位置で相対比較できれば数値ラベル不要
  • 削除した情報 (続2): promise box (sage accent box) を削除し、「記録を続けると…パーソナライズ」をsubtitleに統合。装飾を減らし、本文として自然に伝える
  • 削除した情報 (続3): PFCタグ説明の脚注を削除。凡例 (P/F/C) + タグ自体のラベル (C多め/P主体等) で自明と判断
  • ※ 注釈: infographic下に3行の脚注 (●とバー、よく食べる基準、長押し)。最小限のメタ情報のみ残す
  • 0–300 kcal scale: 食材タブ用。dish タブは別画面で 0–1500 kcal scale を想定
  • おやつバー: 点線で「長押し」表現。modal特定不能を視覚化
  • colors: sage green系 (PRD「静かな日本ウェルネス」トーン)
  • PFC tag chip: 各bucketの default Identity の主成分 (kcal寄与比から判定) を1つのカラータグで表示。design-system tokens の rose/cinnamon/olive (50/700) 系をベースに、複数主成分の場合は2-3色グラデで区別 (P+F=ピンク/茶のhalf, バランス=3色)
  • tour progress: 上部に3点ドット表示 (Stop 1/3)
  • ナビ: スキップ可、次へで Stop 2 へ

⚠ 検討項目

  • infographicが縦に長め — もう少しコンパクト化検討余地
  • 各row下の「ご飯1杯/食パン1枚」表記、長くなるbucketは2行折返し許容するか
  • スケールheader (0/300 kcal) のラベル位置、bar基準点と揃え必要
  • 料理タブ用の Tour Stop 1 別画面 (tab切替時に出すか統合か)
  • PFCラベル文言: "C多め/P主体/P+F/バランス/C少なめ/F のみ/F+C" — 統一感やトーン調整余地

🚀 将来検討

  • パーソナライズ単推し値 (食材 × 量):
    • 食材: ユーザーの長押し履歴 (どのIdentityをよく明示選択するか) を蓄積し、bucketのdefault Identity をユーザー個別に再選定。例: パン派ユーザーは主食 default = bread になる
    • 量: ユーザーの amount 入力履歴を蓄積し、各Identityの default amount をユーザー個別に再調整。例: 大盛り派ユーザーは ご飯 default = 220g になる
