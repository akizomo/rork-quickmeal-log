📝 デザインノート

画面 1: コンセプト

  • ヒーローエリア: 9個のボタングリッドをミニチュアで表示。実画面で何を扱うかを視覚で予告
  • 1ボタンに視線誘導 (highlight): 牛乳ボタンに緑枠+拡大効果。「これがボタンの単位」を直感伝達
  • コピー軸: 「料理名・食材名を細かく覚えなくてOK」= 簡素化価値訴求
  • 進捗: 3/3のうち1つ目アクティブ (画面1 → 画面2 → Tour Stop 1)

画面 2: 操作モデル

  • タップ vs 長押し を 2行で並列表示: 比較で記憶定着しやすい
  • ジェスチャ絵文字 (👆 / ✋): 操作を視覚化。文字より直感的
  • 結果カード: タップ=「ご飯1杯234kcal」/ 長押し=「種類・量を選択」を具体的に提示
  • 下部 tip: 「ふだんはタップ、違う時は長押し」の使い分けルールをそのまま明示
  • CTA: 「はじめる」で直接 メイン画面へ → 自動でTour Stop 1 起動

🔗 オンボ全体フロー

  • 画面1 (コンセプト) → 画面2 (操作モデル) → [はじめる] → メイン画面 → 自動的に Tour Stop 1 (精度infographic) → Stop 2 → Stop 3 → 完了
  • 各画面でスキップ可能。スキップした場合も Tour は表示される (重要な精度情報を確実に1度は見せる)
  • Tour 完了後、初回タップで coachmark (代表値補強メッセージ) が出る

⚠ 検討項目

  • 画面1の hero illustration: ボタングリッドそのまま vs もう少しイラスト寄り (アイコン+「タップ」アニメ風) のどちらがインパクト強いか
  • 画面2 の絵文字 (👆 ✋) を SF Symbols / カスタムアイコンに置き換えるかは実装時判断
  • 「3秒で記録」の数字根拠: テレメトリで実測してから明示するか、感覚値として書き続けるか
