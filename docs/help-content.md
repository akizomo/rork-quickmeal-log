## ライティング原則 (このページに適用)

1. **正確** — IA-identity-spec.md (v1.2) の挙動と一致させる。誇張・過小評価しない
2. **具体** — 「だいたい」「色々」より、可能なら数値や代表名で
3. **詩的常体** — 「いける」「ふみこめる」「残せる」など (intro.tsx と語感統一)
4. **専門用語回避** — modal-set / Identity / PFC収束 などUI用語は使わない
5. **ふだん感** — 「ふだん」「いつも」など日常語を多用
6. **動詞は能動的** — 「記録される」より「記録します」「残せます」
7. **タイトル12字以内、本文28字目安で改行**

## §1. 基本操作

### タイトル
**操作の基本**

### 本文
タップと長押し、2つだけ覚えれば使えます。

### サブセクション

#### タップ
ボタンを押すと、ふだんよく食べる代表値で記録します。
たとえば「ごはんパン麺」をタップ → ご飯1杯 234kcal。

#### 長押し
ボタンを長く押すと、食材の種類や量を細かく選べます。
パンや麺の日、大盛りの日はこちらを使ってください。

### 補足
> ふだんの食事はタップで足ります。
> 違うものを食べた日や、しっかり記録したい日だけ長押しを。

## §2. 食材ボタンの中身

### タイトル
**食材ボタンの中身**

### イントロ
各ボタンには、日本人がよく食べる食材を集約しています。
タップで記録される値は、実際の食事に近い範囲に収まる設計です。

### Infographic
(Tour Stop 1 で作った食材タブ infographic 流用)

| 凡例 | P (たんぱく質) / F (脂質) / C (炭水化物) |

| ボタン (実表示) | 範囲 (kcal) | 主成分 | タップで記録 |
|---|---:|:---:|---|
| 🍚 ごはんパン麺 | 158–234 | C 多め | ご飯1杯 (234kcal) |
| 🐓 肉魚(低脂) | 75–135 | P 主体 | 鶏むね 100g (105kcal) |
| 🥚 卵 | 75 | P+F | 卵1個 (75kcal) |
| 🥩 脂あり肉魚 | 200–230 | P+F | 鶏もも 100g (200kcal) |
| 🥛 乳・大豆 | 62–134 | バランス | 牛乳200ml (134kcal) |
| 🥦 野菜 | 14–55 | C 少なめ | サラダ100g (25kcal) |
| 🍎 果物 | 50–135 | C 多め | バナナ1本 (86kcal) |
| 🧈 油・調味 | 75–110 | F のみ | オイル大さじ1 (110kcal) |
| 🍩 おやつ甘飲 | (長押し) | F+C | — 種類で大きく変わるため毎回選択 |

### 脚注 (※)
- ※ **●** はタップ時に記録される代表値、**バー** は日本人がよく食べる食材の幅です。
- ※ **「よく食べる」基準**: 国民健康・栄養調査 (NHNS) における各バケット内の摂取量シェア上位の食材を採用しています。
- ※ **「長押し」表記** のボタンは、種類により値が大きく変わるため、毎回明示選択する仕様です。

一皿料理は、構成された料理単位の代表値で記録します。

| 🍜 ラーメン | (長押し) | C+F | — 種類で大きく変わる |
| 🍲 うどん蕎麦 | 460–510 | C 主体 | うどん (510kcal) |
| 🍝 パスタ | 620–780 | C+F | トマト系 (680kcal) |
| 🍣 寿司 | (長押し) | C 多め | — 量で大きく変わる |

## §4. もっと、あなたに合わせて (将来予定)

### タイトル
**もっと、あなたに合わせて**

### サブタイトル / バッジ
🚧 今後のアップデートで対応予定

### 本文
記録を続けるほど、よく食べる食材と量に合わせて代表値が調整されていきます。

#### 食材
パンを毎朝食べる方は、「ごはんパン麺」のタップ値がパンに寄ります。

#### 量
大盛り派の方は、ご飯の量も自動で合わせられます。

### 補足
> 今は標準的な日本人の摂取量を基準にしていますが、
> 続けるほど、あなた自身の食習慣に近づきます。

## §5. よくある質問 (FAQ)

### Q1. パンや麺を食べる日はどうすれば?

A. 「ごはんパン麺」ボタンを**長押し**すると、パン・オートミール・餅などから選べます。麺類は「うどん蕎麦」「ラーメン」「パスタ」のボタンに分かれているので、そちらをタップしてください。よく使うものから順に並ぶので、繰り返すうちに早くなります。

---

### Q2. 量がぴったりじゃないけど大丈夫?

A. 標準的な1人前で記録します。大盛り・小盛りはタップ後の画面で切り替えられます。微調整より「続けやすさ」を優先した設計です。

---

### Q3. 1ボタンに何種類も入っていて、ざっくりすぎない?

A. 各ボタンに含まれる食材は、よく食べる範囲ならタップ値とほぼ同じカロリー範囲に収まる設計です。ボタンごとの幅は「**食材ボタンの中身**」「**料理ボタンの中身**」で確認できます。

---

### Q4. 写真で記録できないの?

A. 撮影 → 認識 → 確認の手間で、結局時間がかかり続かない原因になりやすいため、あえてタップ式にしています。3秒で終わるので、毎日続けやすい設計です。

---

### Q5. もっと厳密に計算したい場合は?

A. このアプリは「**ざっくり、続ける**」を優先した設計です。1g単位で計算したい方は、食品成分表ベースの計測アプリ (あすけん等) との併用がおすすめです。

## システム整合チェックリスト (実装時参照)

- ☐ §2 表の **タップで記録** 値は `INGREDIENT_IDENTITIES_BY_BUCKET[bucket][0].defaultMacro.kcal` と一致
- ☐ §2 表の **範囲** はそのバケットの modal-set Identity の min/max kcal
- ☐ §2 「主成分」タグは default Identity の P/F/C カロリー寄与比から算出 (intro.tsx と同じ判定)
- ☐ §3 表も同様 (DISH_IDENTITIES_BY_BUCKET)
- ☐ おやつ・中華麺・寿司・ピザ・定食汁は `bucket.quickTapDisabled === true` のため「長押し」表記で統一
- ☐ §1 例の「ご飯1杯 234kcal」は `getIdentity('rice').defaultMacro` と一致 (現在 234kcal)
- ☐ §1 例の値が変わったら自動追従するよう、UI側は data-driven にレンダリング (ハードコードしない)

## 別タスク: 命名見直し議論 (v3 で検討予定)

現在のボタンラベルは PRD §1.1⑤ の「専門用語排除: 「低脂P」ではなく「鶏むね魚」のような直感ラベル」原則と一部不整合:

- `肉魚(低脂)` — "(低脂)" が栄養学用語
- `脂あり肉魚` — 「脂あり」がやや専門的
- `おやつ甘飲` — "甘飲" は造語っぽい
- `定食・単品・汁` — 長く、「汁」が独立カテゴリと誤解される可能性

**改名の方向性候補** (要議論):

| 現状 | 候補1 | 候補2 |
|---|---|---|
| 肉魚(低脂) | 鶏むね魚 | ヘルシー肉魚 |
| 脂あり肉魚 | 牛豚と脂魚 | 脂多めの肉魚 |
| おやつ甘飲 | おやつ・飲料 | おやつ |
| 定食・単品・汁 | 定食と単品 | 定食ほか |

これは v3 で別途取り扱う想定。本ヘルプページ実装は v2 (現行ラベル準拠) で進める。
